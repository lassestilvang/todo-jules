💡 **What:** Expanded the tap target area for the task completion checkbox by wrapping the task name in an associated `<label>`.

🎯 **Why:** The default 16x16px checkbox is a very small tap target, which can be difficult for users with motor impairments to hit accurately on mobile and desktop (violates WCAG 2.5.5 minimum target size of 44x44px). Making the text itself clickable provides a much more forgiving and satisfying interaction.

📸 **Before/After:** No visual changes other than the cursor changing to a pointer on hover, but functionally it makes completing tasks vastly easier.

♿ **Accessibility:**
- Increased hit area for the completion toggle.
- Added explicit `aria-label` to the Checkbox component for improved screen reader context.
- Semantically linked the task name to its checkbox using `htmlFor` and `id`.
