💡 What:
Refactored the `PUT /api/tasks/[id]` endpoint to use synchronous operations within the `better-sqlite3` database transaction.

🎯 Why:
In Drizzle ORM with `better-sqlite3`, transactions (`db.transaction`) must be strictly synchronous. The previous implementation passed an `async` callback and used `await` inside the transaction block, which breaks transaction bounds and leads to sequential microtask execution overhead (or `TypeError: Transaction function cannot return a promise` errors in strict environments).

📊 Impact:
Removes asynchronous microtask overhead for each query executed within the transaction loop, executing them synchronously at the native C++ add-on speed of SQLite, drastically improving the throughput of bulk subtask updates.

🔬 Measurement:
1. Trigger a subtask update using `PUT /api/tasks/[id]`.
2. Notice the response time decreases significantly as the database engine isn't awaiting promises.
3. Tests (`pnpm test`) assert correct mutation without `TypeError`.
