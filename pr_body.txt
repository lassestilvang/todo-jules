💡 What: Wrapped `SortableTaskItem` inside `src/components/lists/task-list.tsx` using `React.memo()`.

🎯 Why: During drag-and-drop operations, the `optimisticTasks` array is reordered via `arrayMove()`. This causes a full re-render of the parent `TaskList` component. Without `React.memo()`, every single list item is forced to re-render, creating new DOM refs and unnecessarily executing the `useSortable` hook for every item, even though only the dragged item and the shifted items actually changed position. Because `arrayMove` preserves exact object references within the array, memoizing the item component allows React to skip rendering the unaffected items.

📊 Impact: Reduces unnecessary React component tree reconciliations and DOM operations from O(N) to O(1) during drag-and-drop operations. This significantly smooths out drag animations and prevents main thread blocking, particularly noticeable for large task lists.

🔬 Measurement: Use React Developer Tools Profiler. Record a drag-and-drop operation with >100 tasks. Observe that only the actively moved items are re-rendered, rather than the entire list.
