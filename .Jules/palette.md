## 2024-03-20 - [Expand Clickable Area with Labels]
**Learning:** In list-based applications with checkboxes, the default 16x16px checkbox is too small of a tap target, violating WCAG 2.5.5 Target Size (minimum 44x44px). By wrapping the task name in a `<label>` and explicitly associating it with the `<input>` via `htmlFor`/`id`, we dramatically expand the tap target for users with motor impairments while simultaneously providing semantic context for screen readers.
**Action:** Always wrap text adjacent to a checkbox in a `<label htmlFor="id">` and add `cursor-pointer hover:text-primary` to provide visual feedback that the text itself is interactive.
## 2024-03-21 - [Structured Empty States]
**Learning:** Plain "No items found" text strings leave users feeling stuck and make the application feel unpolished. A structured empty state acts as an implicit guide, turning a dead-end into an opportunity for engagement. By combining a large subdued icon, a clear heading, and helpful instructional text within a distinct container (e.g., dashed border), users immediately understand the state and what action to take next.
**Action:** Replace plain empty text strings with structured empty states: use a dashed container (`border-2 border-dashed rounded-lg p-12`), a large subdued icon (opacity-20 or text-muted), a clear heading (`h3`), and subtext that explicitly suggests the next step (e.g., "Get started by adding a new task below.").
